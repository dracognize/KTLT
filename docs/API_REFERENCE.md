# File-by-File API Reference

This document provides a comprehensive list of every source and header file in the KTLT Banking Terminal project and its primary purpose.

## Root Directory
- **`CMakeLists.txt`**: Project build configuration, dependency management (FTXUI, ASIO, Threads), and target definitions (`app`, `server`, `generate_test_data`).
- **`.clang-format`**: Code formatting rules for consistent style across the codebase.
- **`data.db`**: Binary database file storing account records (generated by the server).

## Application Module (`app/`)
- **`app/main.cpp`**: Entry point for the client application. Initializes and runs the `App` class.
- **`app/App.hpp`**: Header for the main application class.
- **`app/App.cpp`**: Implementation of the TUI layout, tab navigation, and global keyboard event handling.

### Client Logic (`app/client/`)
- **`app/client/Client.hpp`**: Header for the networking client. Defines the asynchronous API for server communication.
- **`app/client/Client.cpp`**: Implementation of the ASIO-based client, including the request queue and response dispatcher.

### UI Pages (`app/pages/`)
- **`app/pages/LoginPage.hpp` / `.cpp`**: The login interface and authentication logic.
- **`app/pages/SignupPage.hpp` / `.cpp`**: The account registration interface.
- **`app/pages/DashboardPage.hpp` / `.cpp`**: The main user dashboard showing balance and recent activity.
- **`app/pages/HistoryPage.hpp` / `.cpp`**: Detailed transaction history viewer.
- **`app/pages/DepositPage.hpp` / `.cpp`**: Interface for depositing funds.
- **`app/pages/WithdrawPage.hpp` / `.cpp`**: Interface for withdrawing funds.
- **`app/pages/TransferPage.hpp` / `.cpp`**: Interface for transferring money to other users.
- **`app/pages/SettingPage.hpp` / `.cpp`**: User settings (password change, account locking).
- **`app/pages/AccountsettingsPage.hpp` / `.cpp`**: (Duplicate/Legacy) Additional account settings.

## Base Library (`libs/base/`)
- **`libs/base/algorithm.hpp`**: Generic algorithm implementations (concepts-based).
- **`libs/base/array.hpp`**: Fixed-size array container.
- **`libs/base/forward_list.hpp`**: Singly-linked list container.
- **`libs/base/hash_map.hpp`**: Hash-based associative container (Map).
- **`libs/base/hash_set.hpp`**: Hash-based associative container (Set).
- **`libs/base/hash_table.hpp`**: Core hash table implementation used by Map and Set.
- **`libs/base/hash.hpp`**: Standard hash functions for common types.
- **`libs/base/list.hpp`**: Doubly-linked list container.
- **`libs/base/map.hpp`**: Tree-based associative container (ordered Map).
- **`libs/base/rbtree.hpp`**: Red-Black Tree implementation (balanced BST).
- **`libs/base/set.hpp`**: Tree-based associative container (ordered Set).
- **`libs/base/sha256.hpp` / `.cpp`**: SHA-256 cryptographic hashing implementation.
- **`libs/base/string.hpp`**: Custom string class with SSO optimizations.
- **`libs/base/types.hpp`**: Platform-independent type aliases (u8, i32, usize, etc.).
- **`libs/base/vector.hpp`**: Dynamic array container.

## Network Library (`libs/network/`)
- **`libs/network/Packet.hpp`**: Header for the packet-based networking protocol.
- **`libs/network/Packet.cpp`**: Implementation of synchronous and asynchronous send/recv functions.
- **`libs/network/PacketType.hpp`**: Enum defining all supported packet types in the protocol.

## Server Module (`server/`)
- **`server/main.cpp`**: Entry point for the server application. Initializes the `Server` on a specified port.
- **`server/Server.hpp`**: Header for the `Server` and `Session` classes.
- **`server/Server.cpp`**: Implementation of TCP acceptance and client session handling.
- **`server/Database.hpp`**: Header for the `DbManager` and binary record structures.
- **`server/Database.cpp`**: Implementation of the asynchronous, file-based database engine.
- **`server/ServerLog.hpp`**: Header for the server's console logging utility.
- **`server/ServerLog.cpp`**: Implementation of formatted thread-safe logging.

## Tests and Utilities (`tests/`)
- **`tests/generate_test_data.cpp`**: A utility program to populate the database with initial test accounts and transactions.
